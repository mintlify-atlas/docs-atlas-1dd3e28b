---
title: "Inicio rápido"
description: "Cómo desplegar y configurar Consultoría Online para empezar a recibir respuestas."
---

Esta guía te lleva desde cero hasta una instalación funcional de Consultoría Online en menos de 30 minutos.

## Prerrequisitos

Antes de empezar, asegúrate de tener:

- **Windows Server** con IIS instalado (con el módulo URL Rewrite)
- **.NET Framework 4.7.2** instalado en el servidor
- **SQL Server** (cualquier edición: Express, Standard, Enterprise) accesible por TCP/IP
- **Visual Studio 2019/2022** o **MSBuild** para compilar el proyecto
- Acceso a un **servidor SMTP** para el envío de emails
- **DevExpress** instalado (necesario para la generación de informes DOCX)

<Steps>

<Step title="Obtener el código fuente">

Clona o descarga el repositorio:

<CodeGroup>

```bash git
git clone https://github.com/chaprot/ConsultoriaOnline.git
cd ConsultoriaOnline
```

```powershell PowerShell
Invoke-WebRequest -Uri "https://github.com/chaprot/ConsultoriaOnline/archive/refs/heads/main.zip" -OutFile "ConsultoriaOnline.zip"
Expand-Archive -Path "ConsultoriaOnline.zip" -DestinationPath "."
```

</CodeGroup>

</Step>

<Step title="Crear la base de datos SQL Server">

Crea una nueva base de datos llamada `ConsultoriaOnline`:

```sql
CREATE DATABASE ConsultoriaOnline;
GO
USE ConsultoriaOnline;
GO
```

Crea un usuario de base de datos con permisos mínimos:

```sql
CREATE LOGIN WebAppUser WITH PASSWORD = 'TuContraseñaSegura123!';
CREATE USER WebAppUser FOR LOGIN WebAppUser;
GRANT SELECT, INSERT, UPDATE, DELETE ON SCHEMA::dbo TO WebAppUser;
```

<Warning>
No uses las credenciales de ejemplo del repositorio en producción. Crea un usuario dedicado con contraseña propia.
</Warning>

Crea las tablas necesarias y siembra los datos iniciales (ver [Configuración de la base de datos](/deployment/database-setup) para el esquema completo).

</Step>

<Step title="Configurar la cadena de conexión">

Abre `Misc/DbConnect.cs` y actualiza las cadenas de conexión con tus datos:

```csharp DbConnect.cs
public static string DevXConnString = 
    "data source=TU_SERVIDOR,PUERTO;user id=WebAppUser;password=TuContraseña;initial catalog=ConsultoriaOnline;Persist Security Info=true";

public static string SqlConnString = 
    @"Server=TU_SERVIDOR,PUERTO; Database=ConsultoriaOnline; User=WebAppUser; Password=TuContraseña;";
```

El formato de servidor incluye la IP/hostname y el puerto separados por coma: `185.254.207.112,28919`.

<Note>
`DbConnect.cs` contiene las cadenas de conexión utilizadas por la aplicación en tiempo de ejecución. La cadena en `Web.config` es utilizada exclusivamente por herramientas DevExpress de diseño.
</Note>

</Step>

<Step title="Configurar SMTP">

La configuración SMTP actual está en `Formulario.aspx.cs` dentro del método `EnviarMail`. Actualiza los valores con tu servidor:

```csharp Formulario.aspx.cs
SmtpClient smtpSender = new SmtpClient();
smtpSender.Host = "tu.servidor.smtp.com";
smtpSender.Credentials = new System.Net.NetworkCredential("tu@email.com", "tuContraseña");
smtpSender.Port = 587;
smtpSender.EnableSsl = true;
```

También actualiza la dirección de envío:

```csharp
msg.From = new MailAddress("noreply@tudominio.com", "Tu Empresa");
msg.Bcc.Add("noreply@tudominio.com");
```

<Tip>
Se recomienda mover la configuración SMTP a `Web.config` en la sección `<system.net><mailSettings>` para mayor seguridad. Ver [Configuración SMTP](/deployment/smtp-setup) para más detalles.
</Tip>

</Step>

<Step title="Actualizar el modelo de cuestionario">

En `Formulario.aspx.cs`, el ID de modelo de cuestionario está configurado como `9`:

```csharp Formulario.aspx.cs
int modelo = 9;
```

Asegúrate de que exista un registro con `IdModeloCuestionario = 9` en la tabla `ModeloCuestionario`, junto con sus secciones (`ApartadoPreguntas`), preguntas (`Pregunta`) y respuestas posibles (`RespuestaPosible`).

</Step>

<Step title="Compilar el proyecto">

<CodeGroup>

```bash MSBuild
msbuild ConsultoriaOnlinePrev.sln /p:Configuration=Release
```

```bash Visual Studio
# Abrir ConsultoriaOnlinePrev.sln en Visual Studio
# Menú: Build > Publish Web App
# Configuración: Release
```

</CodeGroup>

</Step>

<Step title="Publicar en IIS">

1. En el **Administrador de IIS**, crea un nuevo sitio web o usa uno existente.
2. Apunta la ruta física a la carpeta de publicación del proyecto.
3. Configura el grupo de aplicaciones con **.NET CLR v4.0** y modo integrado.
4. Asegúrate de que el documento predeterminado esté configurado en `Formulario.aspx`:

```xml Web.config
<defaultDocument>
  <files>
    <clear/>
    <add value="Formulario.aspx"/>
  </files>
</defaultDocument>
```

5. Instala el **módulo URL Rewrite** si no está disponible (necesario para la redirección HTTPS).
6. Concede permisos de escritura al usuario de IIS en las carpetas `~/informes/` y `~/logs/`.

</Step>

<Step title="Abrir en el navegador">

Accede a la URL de tu sitio. Deberías ver el formulario de registro:

- **Formulario de registro** — campo de email y casilla de política de privacidad
- **Mensaje de bienvenida** — tras el registro, con el botón "¡Empezar con el estudio!"
- **Cuestionario** — preguntas secuenciales con barra de progreso y botones de navegación
- **Mensaje de finalización** — barra al 100% y mensaje de cierre

</Step>

<Step title="Registrarse y completar el cuestionario">

1. Introduce una dirección de correo electrónico válida en el campo **Correo electrónico**.
2. Marca la casilla **He leído y acepto la Política de Privacidad**.
3. Pulsa **Regístrarme**. Recibirás un correo de confirmación.
4. Pulsa **¡Empezar con el estudio!** para iniciar el cuestionario.
5. Responde cada pregunta y avanza con **Pregunta siguiente**. Puedes retroceder con el botón **Pregunta anterior**.
6. Al responder la última pregunta, la plataforma envía automáticamente los correos de cierre y muestra el mensaje de finalización.

<Check>
Si recibes el correo de registro y el cuestionario se muestra correctamente, la instalación es exitosa.
</Check>

</Step>

</Steps>
